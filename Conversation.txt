NOCTVM Tech Stack Proposal
The Big Decision: Native App vs Web-First
Your strategy doc spec'd React Native + Expo, and that's still the right call. Here's why, and what's changed since:
Recommendation: Expo (SDK 55+) with Expo Router v5
True native performance on iOS + Android + web from one codebase
File-based routing (like Next.js but mobile-first)
React Server Functions for auth
NativeWind (TailwindCSS for RN) gives you the dark glassmorphism aesthetic natively
The New Architecture is now mandatory in SDK 55 -- no more bridge bottlenecks
Expo EAS handles builds, OTA updates, and submissions to both app stores
Flutter actually wins on raw photo rendering performance (2x faster startup, zero dropped frames), but Expo wins on time-to-market if you know JS/TS, and the ecosystem for social features (image feeds, gestures, animations via Reanimated 3) is mature.
However -- for Phase 1 launch in Romania, I'd suggest a parallel web app too. Not everyone will download an app from an unknown brand. A Next.js web companion (even stripped down) lets people discover events via Google, share links that open in browser, and lower the friction to first experience. Expo Router v55 supports SSR experimentally, or you run a separate Next.js site sharing the same API.
Backend: Supabase
Clear winner for NOCTVM's use case:
What you need	Why Supabase
Relational data (users, venues, events, follows, posts)	PostgreSQL with full SQL -- your data model is inherently relational
Auth	Email/password, Google, Apple, phone -- all built-in, free up to 50K MAU
Real-time feeds	Row-level subscriptions via WebSockets, filterable with SQL
Row-Level Security	Users see only what they should -- RLS policies tied to auth
Image storage	Built-in with CDN, image transformations on Pro ($25/mo)
Cost	Predictable -- unlimited API requests, no per-read charges (Firebase would kill you on feed reads)
Self-hostable	No vendor lock-in, open source
Convex has better reactivity but smaller ecosystem. Firebase's per-read pricing would bleed you on a feed-heavy app. Supabase's $25/mo Pro plan covers 8GB database, 100GB storage, and all the auth/realtime you need through launch.
Media Pipeline: Cloudflare R2 + Cloudflare Images
Supabase Storage works for MVP, but the moment you scale user-generated photos, you want:
R2: Zero egress fees (vs $0.09/GB on Supabase). For a photo-heavy social app, this saves hundreds/month
Cloudflare Images: On-the-fly resize/format conversion at the edge (thumbnails, WebP/AVIF, quality control)
Hybrid approach: Supabase for authenticated private content (avatars, DMs), R2 for all public venue/event photos
This can be Phase 2 -- start with Supabase Storage, migrate public media to R2 when costs justify it.
